# Переписать MIDI-плеер: надёжный планировщик на Web Audio

## Overview

MIDI/beat-плеер в `components/BeatPlayer.vue` написан на низкоуровневых JS-циклах поверх
`requestAnimationFrame` и заикается: ритм сбивается примерно раз в минуту, тормозит на телефоне,
чувствителен даже к задержке ~50 мс. Цель — переписать движок воспроизведения на надёжный
планировщик по аудио-часам (look-ahead scheduling по `AudioContext.currentTime`), убрать
заикания и сделать переключение инструментов на границе тактов. Все существующие функции
приложения должны сохраниться.

## Context

- Затрагиваемый компонент: `components/BeatPlayer.vue` (движок: `tick()`, `sendNotes()`,
  `startPlay()`, `startLoad()`, петля/синк двух треков `songDrums` + `songPiano`).
- Воспроизведение через WebAudioFont (`player.queueWaveTable`) с точным `when` по
  `AudioContext.currentTime`; барабаны из `assets/beats.json`, пиано — генерируется из аккордов.
- Прогресс пробрасывается в store (`beatProgress`, `beatPlaying`) и используется в
  `ChordsFloating` (мобильный метроном) — контракт со store ломать нельзя.
- Функции, которые обязаны сохраниться: стили пиано, пресеты, октава, sustain, rever,
  drums-only / piano-only / piano-in-drums (`highestPitch >= 60`), кастомные инструменты,
  BPM-слайдер/метр, горячие клавиши (b/c/d и рус. раскладка), чтение параметров из URL.
- Адаптировано из `docs/TODO.md`, раздел «Переписать MIDI-плеер». Переезд на Vue 3 (ветка `v5`)
  в этот план НЕ входит — он отложен и делается после.

## Technical Details

- **Текущая проблема:** `tick()` крутится через `requestAnimationFrame` с окном упреждения
  `stepDuration = 44 мс` и на каждом шаге `sendNotes()` сканирует ВСЕ ноты трека (O(n)).
  RAF зависит от частоты кадров, приостанавливается при перерисовке/в фоне и даёт jitter.
- **Целевой паттерн** — «A Tale of Two Clocks» (Chris Wilson): таймер на `setTimeout` с шагом
  ~25 мс и окном упреждения ~100–150 мс; ноты ставятся в очередь заранее с абсолютным `when` по
  `AudioContext.currentTime`. Аудио-планировщик не зависит от кадров и переживает подвисания UI
  в пределах окна упреждения.
- **Петля/синк:** сохранить выравнивание `songStart` по числу проигранных циклов
  (`songBeatsDuration`, коррекция дрейфа), интро-отсчёт перед стартом, смещение под
  `audioContext.outputLatency`, сброс пиано-трека вместе с барабанным на границе цикла.
- **Граница 4 тактов:** `beatCycleDuration = beatDuration * chordBeats * 4` — единица применения
  отложенных изменений (инструмент, стиль, BPM).
